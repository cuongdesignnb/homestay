# 🚨 Fix 403 Forbidden Error - aaPanel/Nginx

## 🔍 **Nguyên nhân thường gặp:**

### 1. **File Permissions (Phổ biến nhất)**

```bash
# SSH vào server hoặc dùng Terminal aaPanel
cd /www/wwwroot/catbacountrysidehomestay.com

# Kiểm tra permissions hiện tại
ls -la backend/public/

# Set permissions đúng
chmod -R 755 backend/
chmod -R 755 backend/public/
chown -R www:www backend/
chown -R www:www backend/public/

# Hoặc nếu dùng www-data
chown -R www-data:www-data backend/
```

### 2. **Document Root sai**

Trong aaPanel → Website → yoursite.com → Settings → Website Directory:

- **Phải là:** `/www/wwwroot/catbacountrysidehomestay.com/backend/public`
- **KHÔNG phải:** `/www/wwwroot/catbacountrysidehomestay.com/`

### 3. **Index file không tồn tại**

```bash
# Kiểm tra file index
ls -la backend/public/index.php
ls -la backend/public/index.html

# Nếu thiếu, tạo symlink
cd backend/public/
ln -sf ../../../frontend/dist/index.html ./index.html
```

## 🔧 **Bước xử lý:**

### **Bước 1: Kiểm tra Document Root**

1. Vào **aaPanel → Website → catbacountrysidehomestay.com**
2. Click **Settings**
3. Tab **Website Directory**
4. Đặt: `/www/wwwroot/catbacountrysidehomestay.com/backend/public`
5. **Save**

### **Bước 2: Fix Permissions**

```bash
# Trong Terminal aaPanel
cd /www/wwwroot/catbacountrysidehomestay.com

# Set owner
chown -R www:www ./
# Hoặc nếu lỗi thử:
chown -R www-data:www-data ./

# Set permissions
find . -type d -exec chmod 755 {} \;
find . -type f -exec chmod 644 {} \;

# Laravel specific
chmod -R 775 backend/storage/
chmod -R 775 backend/bootstrap/cache/
```

### **Bước 3: Nginx Configuration**

Vào **aaPanel → Website → Settings → Config → Nginx** và thêm:

```nginx
location / {
    try_files $uri $uri/ /index.html;
}

location /api {
    try_files $uri $uri/ /index.php?$query_string;
}

location ~ \.php$ {
    fastcgi_pass unix:/tmp/php-cgi-82.sock;  # Hoặc php version bạn dùng
    fastcgi_index index.php;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    include fastcgi_params;
}

# Disable access to sensitive files
location ~ /\. {
    deny all;
}

location ~ /(vendor|storage|bootstrap|config|database|resources|routes|tests)/ {
    deny all;
}
```

### **Bước 4: Check PHP Version**

1. **aaPanel → Website → Settings → PHP Version**
2. Chọn **PHP 8.2**
3. **Extensions** → Enable: `fileinfo`, `mbstring`, `openssl`, `pdo_mysql`, `tokenizer`

### **Bước 5: Verify Files**

```bash
# Check Laravel entry point
curl -I http://catbacountrysidehomestay.com/api/

# Check frontend
curl -I http://catbacountrysidehomestay.com/

# Check permissions
ls -la /www/wwwroot/catbacountrysidehomestay.com/backend/public/
```

## 🔍 **Debug Commands:**

### **Check Nginx Error Log:**

```bash
tail -f /www/server/nginx/logs/error.log
```

### **Check PHP-FPM Log:**

```bash
tail -f /www/server/php/82/var/log/php-fpm.log
```

### **Test file access:**

```bash
# Test if nginx can read files
sudo -u www cat /www/wwwroot/catbacountrysidehomestay.com/backend/public/index.php
```

## 🎯 **Quick Fix Commands:**

```bash
# Run these in order:
cd /www/wwwroot/catbacountrysidehomestay.com

# 1. Fix ownership
chown -R www:www ./

# 2. Fix permissions
chmod -R 755 ./
chmod -R 775 backend/storage backend/bootstrap/cache

# 3. Restart services
systemctl reload nginx
systemctl restart php-fpm-82

# 4. Test
curl -I http://catbacountrysidehomestay.com/
```

## ⚠️ **Nếu vẫn lỗi:**

1. **Check SELinux** (nếu có):

   ```bash
   getenforce
   # Nếu = Enforcing, tạm thời disable:
   setenforce 0
   ```

2. **Check Firewall:**

   ```bash
   ufw status
   iptables -L
   ```

3. **Recreate site** trong aaPanel với cấu hình mới
